  • Instructions for using the app
    • This uses a 2-key interface. To curse through the buttons, press the a. To select an item, press the enter button.
  • Test results
    • For the tests, I used the following equations:
    754 / 28 - 2
    3 + 61 - 0
    99 * 44 / 2 + 1
    These are the equations that I chose because they require pressing all the buttons, and require you to go back around again with the cursor (the 2-key has no goBack button).
    I ran this test with hand vs. elbow vs. foot. I ran each body part 3 times. The average times were:
    hand: 1 min 22 sec. average: 2 mistakes
    elbow: 1 min 47 sec. average: 4 mistakes
    foot: 2 min 10 sec. average: 4 mistakes
    From these results, I have concluded that the hand had the best results. This is most likely due to the fact that I have more control over my hand compared to my foot and elbow.
    However, some people may have better control over other parts of the body, so it was good to test to make sure it works with other body parts.
    The most common mistake was using the cursor and scrolling past the number you needed to select, because then you had to go through and cursor over everything again before it gets back to the item you missed.
  • Anything else you have tried out
    • At first, I was using the keys "a" for scroll and "b" for select. However, I found this problematic because, with the other body parts, it was hard to select the smaller buttons.
    So, I changed the buttons to "spacebar" is scroll and "enter" is select. Since these buttons are bigger in size, it was easier to select these buttons with my elbow and foot.
    However, when I ran this code on Github, the spacebar did not work because it just scrolled the page down, so I had to switch it back to the a button to curse/scroll, and still enter button to select.
